1. tính logic của  !(true / fale)

giả sử ta có đoạn code sau:

for(int i = 0, k = 0; i < spl.length; i++) {

	int uniqueCount = 0;

            boolean isExist = false;
            for(int j = 0; j < uniqueCount; j++) {

                if(spl[i].equalsIgnoreCase(unique[j])) {
                    isExist = true;
                    break;
                }
            }

            

            if(!isExist) {
                unique[k++] = spl[i];
                uniqueCount++;
            }
        }
        

đoán xem liệu biến uniqueCount có tăng lên hay không??

đáp án là có

ở đoạn code trên, vì ta lầm tưởng rằng điều kiện j < uniqueCount cho vòng for không
thỏa mãn nên isExist vẫn là false, từ đó dòng if không chạy vì không thỏa mãn
điều kiện !isExist, từ đó uniqueCount++ không thực thi và uniqueCount không tăng lên
 nhưng mọi câu trả lời nằm ở điều kiện !isExist.

ở đây ta phải hiểu tính logic của điều kiện của dòng if(!isExist) 

nếu isExist là false thì !isExist trả về true.

và if(!isExist) có nghĩa là if ( !isExist == true) . nói cách khác thì điều kiện này 
luôn hoạt động dù cho vòng lặp không diễn ra. khi đó uniqueCount++
được thực thi và biến uniqueCount tăng lên cho dù isExist là false . Mấu chốt ở đây là
ta hiểu nhầm rằng if(!isExist) nghĩa là "nếu isExist khác false thì chạy lênh" . 

2. tính logic của vòng lặp for 

giả sử ta có vòng lặp for như sau:
for (int i = 0; i < 5; i++) {
    if (i == 3) {
        break;
    }
}

Trong vòng lặp for, nếu điều kiện không thỏa mãn thì khối lệnh bên trong
 vòng lặp sẽ không được thực thi và bước cập nhật biến đếm (i++ trong trường hợp này) cũng sẽ không được thực hiện.

Nói cách khác, nếu điều kiện trong vòng lặp for không thỏa mãn ngay từ 
đầu (tức là for không bao giờ được thực thi), thì biến đếm i sẽ giữ nguyên giá trị của nó ban đầu mà không tăng lên.
